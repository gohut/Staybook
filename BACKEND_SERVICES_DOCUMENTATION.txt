Staybook Backend Services Documentation
======================================

Generated from the current repository state.

1) Service Overview
-------------------
- auth-service
  - Port: 8081
  - Database: MySQL (jdbc:mysql://localhost:3306/staybook_auth)
  - Purpose: authentication, user registration, JWT issuance

- user_profile_service
  - Port: 8082
  - Database: MongoDB (mongodb://localhost:27017/staybookUserProfile)
  - Purpose: traveler profiles & bookings, partner profiles & hotels, public hotel catalog, admin settings/coupons

- Inventory_service
  - Port: 8085
  - Database: MongoDB (mongodb://localhost:27017, db: staybook_role)
  - Purpose: partner applications & booking inventory
  - File storage: GridFS for partner application documents

2) auth-service
---------------
Tables (SQL)
- users (JPA entity: com.staybook.authservice.entity.User)
  - id (PK), name, email (unique), password, role, createdAt

Endpoints
- POST /api/auth/register
- POST /api/auth/login
- PUT  /api/auth/password

Frontend connections
- frontend/src/Api/authApi.js
  - loginApi() -> /api/auth/login
  - registerApi() -> /api/auth/register

- Used in:

  - Admin_portal_/Hotel_Registration/PartnerCredentialsPop.jsx (create partner login)

5) Cross-Service Connections
----------------------------
- Auth -> User Profile
  - auth-service issues JWT; user_profile_service expects Authorization: Bearer <token> for most endpoints.
  - Partner email in user_profile_service is the JWT subject (auth email).

- Inventory bookings -> Traveler bookings
  - Main website creates a booking in Inventory_service (hotel_booking) and also creates a traveler booking in user_profile_service (traveler_bookings) when logged in.
  - traveler_bookings.bookingId corresponds to inventory_service.hotel_booking.bookingId

- Partner applications -> Admin & Partner creation
  - Inventory_service stores partner_applications + document files (GridFS).
  - Admin portal reviews applications and creates partner auth accounts (auth-service) and partner profiles (user_profile_service).

- Hotel media
  - user_profile_service stores hotel photos in GridFS; hotel_photos.fileId references the file in GridFS.
  - Public and partner endpoints stream the photo bytes from GridFS.
